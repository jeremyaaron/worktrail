The app

A focused operating surface for project work.

Worktrail is intentionally small, but the workflows are real. v0.1.0 consolidates the app around daily execution, and v0.1.1 adds the attention loop: Inbox, work item watchers, comment mentions, and actor-scoped read state. The product stays focused while exercising production-shaped collaboration patterns.

Daily execution

Start from the work that needs attention.

  • My Work daily queue for the selected local actor.
  • Inbox summary for unread collaboration updates.
  • Quick capture for project-scoped work from a global route.
  • Comments, tombstones, activity, and return-to-list context.
Action Inbox

Route important changes to the right person.

  • Unread and all notification views for the selected actor.
  • Read, unread, and mark-all-read controls.
  • Deep links back to mentioned, assigned, watched, and blocked work.
Work discovery

Find the right work across projects.

  • Top-level Work Items with URL-backed filters and active chips.
  • Personal saved views with compact management controls.
  • Filtered CSV export for project and workspace result sets.
Project operation

Keep every project view in context.

  • Persistent project shell with status and health.
  • Overview, Work, Board, Planning, and Settings subnav.
  • Durable Angular CDK board order with server validation.
Planning

Review health before editing administration.

  • Planning Review with needs attention, upcoming, and recent changes.
  • Milestone progress with explainable health reasons.
  • Risk sections for overdue, blocked, stale, and unassigned work.
Dependency visibility

Make blockers part of the operating model.

  • Directional blocking and symmetric related-work links.
  • Derived dependency-blocked and blocking-open-work signals.
  • Dependency filters across lists, saved views, planning, and export.
Workspace governance

Exercise role paths without fake SaaS ceremony.

  • Owner, maintainer, and contributor policy paths.
  • Member lifecycle and inactive-member history.
  • Archived projects stay readable and block writes.

v0.1.1 baseline

Product polish, collaboration signals, and reference-app discipline.

The v0.1.0 release turned accumulated capability into a clearer baseline. v0.1.1 adds an actor-scoped collaboration layer: persisted in-app notifications, work item watchers, comment mentions, unread state, and deterministic fan-out from domain commands. Together they make Worktrail feel less like a passive tracker and more like a daily operating surface.

For product users Lower scan cost, clearer navigation, visible collaboration updates, and readable mobile work lists.
For builders Domain contracts, notification fan-out, route-level code splitting, shared query/risk policy, endpoint handlers, and API client boundaries.
For operators Production preview, runtime config validation, health/readiness checks, OpenAPI, local runbook, CI, and deterministic seeds.

Operations

Built artifacts, readiness signals, and documented run paths.

The production preview runs compiled API code and serves the built Angular app from Express. It validates runtime configuration, requires an explicit production DATABASE_URL, serves SPA deep links, keeps API routes separate from browser fallback, and exposes process liveness plus database readiness endpoints. CI now runs lint, typecheck, unit tests, and production build on pull requests and pushes to main.

Preview

One command from source to built runtime.

npm run preview builds contracts, API, and web artifacts, then starts production mode.

Readiness

Separate process and database checks.

/api/health/live confirms the API process; /api/health/ready checks Postgres.

API docs

Runbook

Operational commands with explicit caveats.

Setup, migrations, reset safety, preview, deep-link checks, and cloud mapping are covered.

Architecture

Local-first now. Cloud-shaped from the first cut.

Worktrail runs locally with Angular, Express, and Postgres. The internal boundaries are deliberately boring: shared DTO contracts, request validation, repository-backed services, actor context, command-style writes, and endpoint handlers that can outlive the local Express adapter.

Current scope

Local-first, cloud-shaped, still honest about limits.

Worktrail v0.1.1 is still a local reference app. Production authentication, invitations, workspace-visible saved views, pinned projects, custom roles, custom workflows, email or push delivery, notification preferences, attachments, third-party migration mappings, graph visualization, dependency automation, custom health formulas, forecasting, critical path analysis, saved review snapshots, and hosted infrastructure are intentionally out of scope. Production preview is not an internet-safe authenticated deployment. The value today is a working product surface and operational boundary that can accumulate evidence before patterns are extracted into broader framework ideas.
