The app

A reference workflow with enough product surface to matter.

Worktrail is intentionally small, but it is not a toy shell. Earlier releases cover project settings, milestones, label catalogs, keyed work items, auto-applied discovery filters, a persisted drag/drop board, planning summaries, comment lifecycle controls, local actor selection, and activity events. v0.0.4 adds the team layer: workspace settings, member lifecycle, role-aware affordances, inactive-member handling, and workspace activity so implementation choices have real consequences.

Workspace

Team governance without production auth theater.

Create, edit, deactivate, and reactivate members while preserving a local actor model.

Permissions

Role-aware controls with server enforcement.

Owners, maintainers, and contributors see clear affordances backed by API policy checks.

Projects

Configurable projects with readable lifecycle state.

Create projects with explicit keys, edit metadata, manage archive/reactivate flows, and keep archived projects read-only.

Milestones

Project-scoped planning targets.

Create milestone goals, assign work, track lifecycle state, and keep archived history intact.

Discovery

Search and filters that match everyday triage.

Find work by status, type, priority, assignee, label, milestone, or search text.

Board

Drag/drop movement with durable order.

Move and reorder cards with Angular CDK while server-side workflow validation remains authoritative.

Planning

Dashboard signals for near-term delivery risk.

Review milestone progress, overdue work, blocked work, unassigned work, and stale work.

Collaboration

Comments, tombstones, and activity history.

Add, edit, and delete comments with visible tombstones, lifecycle activity, and inactive-member history.

v0.0.4 capabilities

Enough product depth to exercise real patterns.

The current sprint makes Worktrail governable by a small team: workspace settings, member administration, server-derived actor roles, active/inactive member behavior, permission transparency, workspace activity, project creation polish, and browser-level smoke coverage for the governance path. Planning, discovery, board ordering, comments, and project activity remain part of the same reference surface.

Architecture

Local-first now. Cloud-shaped from the first cut.

The MVP runs locally with Angular, Express, and Postgres. The internal boundaries are deliberately boring: shared DTO contracts, request validation, repository-backed services, actor context, command-style writes, and endpoint handlers that can outlive the local Express adapter.

Angular SPAstatic build suitable for S3 and CloudFront
Lazy routesroute chunks keep initial production bundles below warning budgets
Endpoint handlerstransport-neutral request and response shape
Express adapterlocal runtime only for v0.0.4
Postgres + Drizzlecommitted SQL migrations and deterministic seed data
Playwright smokebrowser-level workflow coverage for release readiness

Current scope

Local-first, cloud-shaped, still honest about limits.

Worktrail v0.0.4 is still a local reference app. Production authentication, invitations, custom roles, custom workflows, notifications, attachments, imports, and hosted infrastructure are intentionally out of scope. The value today is a working product surface that can accumulate evidence before patterns are extracted into broader framework ideas.
