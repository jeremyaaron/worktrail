The app

A reference workflow with enough product surface to matter.

Worktrail is intentionally small, but it is not a toy shell. The MVP covers project spaces, work item lists, a board, comments, labels, local actor selection, and activity events so implementation choices have real consequences.

Projects

Seeded workspaces with project summaries.

Open projects, review status counts, and move from summary to list or board views.

Work items

Structured work with filters and detail pages.

Track type, status, priority, assignee, labels, due dates, estimates, and comments.

Board

Status-driven movement without hidden behavior.

Use explicit status menus backed by server-side workflow validation and activity events.

Activity

A timeline that proves state changed.

Creation, comments, labels, priority, assignee, and status changes are recorded.

Architecture

Local-first now. Cloud-shaped from the first cut.

The MVP runs locally with Angular, Express, and Postgres. The internal boundaries are deliberately boring: shared DTO contracts, request validation, repository-backed services, actor context, and endpoint handlers that can outlive the local Express adapter.

Angular SPAstatic build suitable for S3 and CloudFront
Endpoint handlerstransport-neutral request and response shape
Express adapterlocal runtime only for v0.0.1
Postgres + Drizzlecommitted SQL migrations and deterministic seed data
Playwright smokebrowser-level workflow coverage for release readiness

MVP scope

A public face for an early reference app.

Worktrail v0.0.1 is a local MVP. Authentication, permissions, label administration, hosted infrastructure, and richer collaboration tools are intentionally limited. The value today is a working reference implementation that can accumulate evidence before patterns are extracted into broader framework ideas.
