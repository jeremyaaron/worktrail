The app

A focused operating surface for project work.

Worktrail is intentionally small, but the workflows are real. v0.1.0 consolidates the app around daily execution, v0.1.1 adds the attention loop, v0.1.2 hardens filtered work views, v0.1.3 turns saved views into workspace operating lenses, v0.1.4 brings the same pattern into project workspaces, and v0.1.5 promotes the most important saved views into pinned shortcuts, v0.1.6 adds project batch triage for selected project work, v0.1.7 adds focused milestone review pages, v0.1.8 adds project status reports with immutable published snapshots, and v0.1.9 makes those reports portable with Markdown copy, Markdown download, and print-friendly detail pages.

Daily execution

Start from the work that needs attention.

  • My Work daily queue for the selected local actor.
  • Inbox summary for unread collaboration updates.
  • Quick capture for project-scoped work from a global route.
  • Comments, tombstones, activity, and return-to-list context.
Action Inbox

Route important changes to the right person.

  • Unread and all notification views for the selected actor.
  • Read, unread, and mark-all-read controls.
  • Deep links back to mentioned, assigned, watched, and blocked work.
Work discovery

Find the right work across projects.

  • Top-level Work Items with URL-backed filters and active chips.
  • Copyable filtered-view links that reload to the same applied work set.
  • Personal, workspace-shared, and project-scoped saved views with compact controls.
  • Pinned workspace and project shortcuts for important operating lenses.
  • Contributor read access to shared workspace/project views with owner/maintainer management rules.
  • Filtered CSV export for project and workspace result sets using applied filters.
Project operation

Keep every project view in context.

  • Persistent project shell with status and health.
  • Overview, Work, Board, Planning, and Settings subnav.
  • Durable Angular CDK board order with server validation.
  • Owner/maintainer batch triage for selected project work items.
Planning

Review health before editing administration.

  • Planning Review with needs attention, upcoming, and recent changes.
  • Milestone progress with explainable health reasons.
  • Milestone review pages with scope, risk, and recent movement.
  • Risk sections for overdue, blocked, stale, and unassigned work.
  • Query-backed risk links into filtered project Work.
Status reports

Turn current project state into a durable update.

  • Generated drafts from health, milestone, risk, and recent-work signals.
  • Editable narrative before publication.
  • Immutable published snapshots with links back to current work.
  • Markdown copy/download and print-friendly report detail pages.
  • Owner/maintainer publishing with contributor read access.
Dependency visibility

Make blockers part of the operating model.

  • Directional blocking and symmetric related-work links.
  • Derived dependency-blocked and blocking-open-work signals.
  • Dependency filters across lists, saved views, planning, and export.
Workspace governance

Exercise role paths without fake SaaS ceremony.

  • Owner, maintainer, and contributor policy paths.
  • Member lifecycle and inactive-member history.
  • Archived projects stay readable and block writes.

Operations

Built artifacts, readiness signals, and documented run paths.

The production preview runs compiled API code and serves the built Angular app from Express. It validates runtime configuration, requires an explicit production DATABASE_URL, serves SPA deep links, keeps API routes separate from browser fallback, and exposes process liveness plus database readiness endpoints. CI now runs lint, typecheck, unit tests, and production build on pull requests and pushes to main.

Architecture

Local-first now. Cloud-shaped from the first cut.

Worktrail runs locally with Angular, Express, and Postgres. The internal boundaries are deliberately boring: shared DTO contracts, request validation, repository-backed services, actor context, command-style writes, and endpoint handlers that can outlive the local Express adapter.

Current scope

Local-first, cloud-shaped, still honest about limits.

Worktrail v0.1.9 is still a local reference app. Production authentication, invitations, pinned projects, custom roles, custom workflows, cross-project bulk edit, email or push delivery, notification preferences, attachments, third-party migration mappings, graph visualization, dependency automation, custom health formulas, forecasting, roadmap views, critical path analysis, milestone sign-off, PDF reports, report delivery, report approvals, public report links, export history, and hosted infrastructure are intentionally out of scope. Production preview is not an internet-safe authenticated deployment. The value today is a working product surface and operational boundary that can accumulate evidence before patterns are extracted into broader framework ideas.
