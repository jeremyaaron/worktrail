Project management reference app

Worktrail

Worktrail is a focused project management app for daily execution, project operation, cycle planning, collaboration, dependencies, and published status snapshots.

It is also a reference implementation for an Angular static frontend, a TypeScript API boundary, domain services, Postgres persistence, and production-shaped local operations.

The product

A working operating surface for team project work.

Worktrail is intentionally compact, but its workflows are real. The app gives a team a daily starting point, durable ways to find and shape work, project-level operating views, and published status artifacts that preserve what was true when a report was shared.

Start the day

Open the work and updates that need attention.

  • My Work summarizes assigned work, due dates, dependency-blocked work, and stale items.
  • Action Inbox collects mentions, assignments, watcher updates, and dependency changes.
  • Local actor switching exercises owner, maintainer, and contributor paths.
  • Notifications link back to the relevant work item or project context.
Find and shape work

Use applied filters as a product contract.

  • Workspace and project work lists use URL-backed filters, active chips, and copyable links.
  • Saved views support personal, workspace-shared, and project-scoped operating lenses.
  • Pinned shortcuts keep the most important saved views visible.
  • CSV import, dry-run validation, transactional apply, and filtered exports support portability.
Operate projects

Keep project decisions close to delivery state.

  • Project shell navigation keeps Overview, Work, Board, Planning, Reports, and Settings in context.
  • Angular CDK board interactions preserve durable server-validated order.
  • Owner and maintainer bulk triage applies scoped project updates with per-row results.
  • Relationships surface blocked-by, blocking, and related work across the workspace.
Share status

Separate live planning from published evidence.

  • Planning, Milestone Review, and Cycle Review pages are labeled as live views of current state.
  • Cycles track methodology-neutral execution windows with scoped work and target points.
  • Report drafts separate generated evidence from editable narrative.
  • Published reports preserve immutable milestone, cycle, risk, and recent-work snapshots while linking to current work.
  • Markdown copy, Markdown download, and print-friendly report detail pages support sharing.

Architecture

Local-first now. Cloud-shaped where it matters.

Worktrail runs locally with Angular, Express, and Postgres. The important reference value is in the boundaries: frontend state is explicit, HTTP handlers are adapter-ready, domain rules live in services, persistence is relational, and operations have concrete build and readiness paths.

FrontendAngular standalone route components, lazy route chunks, static build output, URL-backed query state, and responsive work surfaces suitable for S3/CloudFront hosting.
API boundaryShared DTO contracts, explicit request validation, transport-neutral endpoint handlers, Express local adapter, OpenAPI documentation, and non-JSON export responses.
DomainProject planning, cycle review, delivery health, risk sections, relationships, notifications, saved views, CSV flows, report snapshots, and permission paths in service-level code.
PersistencePostgres with Drizzle schema and migrations, deterministic seed data, project cycles, immutable report snapshots, activity records, notifications, relationships, and saved view state.
OperationsProduction preview, runtime config validation, liveness/readiness endpoints, CI lint/typecheck/test/build, and local smoke-test guidance.

Operations

Built artifacts, readiness signals, and documented run paths.

The production preview compiles the API and Angular app, serves built frontend assets from Express, keeps `/api/*` separate from SPA fallback, requires an explicit production database URL, and exposes separate process liveness and database readiness checks.

Preview

One command from source to built runtime.

npm run preview builds contracts, API, and web artifacts, then starts production mode.

Readiness

Separate process and database checks.

/api/health/live confirms the API process; /api/health/ready checks Postgres.

API docs

A checked-in OpenAPI reference.

Implemented routes, local actor headers, representative DTOs, readiness behavior, and errors are documented.

Runbook

Operational commands with explicit caveats.

Setup, migrations, reset safety, preview, deep-link checks, and cloud mapping are covered.

Current scope

Useful reference app, not hosted SaaS.

Worktrail is candid about what it is not. The value today is a working product surface and production-shaped local architecture, not an internet-safe deployment.

No production auth yet Authentication is represented by local actor headers and the top-bar actor selector. Real identity, invitations, tenant provisioning, and custom roles are deferred.
No hosted infrastructure yet The production preview is for local operational inspection. Managed deployment, secrets, observability, background workers, and internet-safe hosting are not included.
No enterprise workflow suite yet Custom workflows, attachments, roadmap forecasting, cycle capacity planning, report approvals, public report links, critical path analysis, and third-party migrations remain outside the baseline.

Run locally

Clone, migrate, seed, and start.

The default local path uses Postgres, deterministic seed data, the Angular dev server, and the local Express API. 

npm install
npm run db:start
npm run db:migrate
npm run db:seed
npm run dev

# production preview
DATABASE_URL=postgres://worktrail:worktrail@localhost:5432/worktrail npm run preview
